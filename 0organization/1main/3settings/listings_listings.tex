% Some Instructions on how to properly format a Listing.
% The Problem to solve:
%  - Standard Keywords are actually by itself properly formatted, like 'main', 'int', etc.
%     -> What is automatically detected as such a keyword depends on the specified language at Listing-Inclusion
%  - But of course, it cannot detect arbitrary functions-names, you defined, or typedefs, ...
% To have them still nicely formatted, I provide two methods (Examples below):
%  1) Just define them as "morekeywords" for their fitting category, on the \lstinputlisting
%  2) Manual formatting. This would be done per word directly in the listing.
%     In principle, you "escape-to-LaTeX" via a special escape-character. In my setup, I defined this to be '¢'. Everything between two of these, is treated as standard LaTeX, so you can format it.
%     To make that easier, use the Commands, defined above.
%------------------------------------------------------------------------
%= = 1)   = = = = = = = =
%    \lstinputlisting[language=DenKr-C,label=lst:ecs,%
%        caption={ECS code snippet},%
%        morekeywords={[1]{void}},%Basic Keywords. If, return, int, void
%        morekeywords={[2]{memcpy}},%Predefined Library-Functions
%        morekeywords={[3]{some_function}},%Function-Calls
%        morekeywords={[4]{PluginManager}},%Typedefed, Names of Struct, Enum, ...
